## Plug-in Architecture —

Plug-in Architecture (ან **მოდულარული არქიტექტურა**) არ არის მარტო კოდის დიზაინის პატერნი — ეს უფრო ფართო **სისტემური არქიტექტურის მიდგომაა**, რომელიც საშუალებას გაძლევს აპლიკაცია ააწყო ისე, რომ მისი ნაწილები (პლაგინები)  
შეიძლება **დამატდეს, შეიცვალოს ან წაიშალოს ბირთვის ცვლილების გარეშე**.

> _“გააფართოვე სისტემა ახალი ქვეკომპონენტებით, მაგრამ არ შეცვალო ძველი.”_


## იდეა (ფილოსოფიურად)**

Plug-in Architecture ეფუძნება პრინციპს:

> **“Core system nothing knows about features — only interfaces.”**

ანუ, სისტემა არის ჩარჩო (Framework/Core), ხოლო ფუნქციონალები (Features) მიერთებულია პლაგინების სახით.  
ეს იდეა ყველგან გვხვდება — WordPress-ის მოდულები, Visual Studio extensions, Unity Editor-ის plug-in-ები, Revit Add-ins, და .NET middleware-ებიც კი.
