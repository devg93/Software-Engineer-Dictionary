рЃФрЃљрЃџрЃўрЃљрЃю рЃЎрЃљрЃарЃњрЃў рЃЏрЃўрЃЊрЃњрЃЮрЃЏрЃљрЃљ Рђћ рЃљрЃ«рЃџрЃљ рЃќрЃБрЃАрЃбрЃљрЃЊ рЃўрЃАрЃћ рЃљрЃњрЃўрЃ«рЃАрЃюрЃў, рЃарЃЮрЃњрЃЮрЃарЃф рЃЊрЃћрЃЋрЃћрЃџрЃЮрЃърЃћрЃарЃўрЃА рЃЊрЃўрЃЦрЃерЃюрЃљрЃарЃерЃў рЃБрЃюрЃЊрЃљ рЃўрЃДрЃЮрЃА рЃљрЃдрЃгрЃћрЃарЃўрЃџрЃў ­ЪњА  
рЃЏрЃљрЃарЃбрЃўрЃЋрЃў рЃћрЃюрЃўрЃЌ, рЃЏрЃљрЃњрЃарЃљрЃЏ рЃбрЃћрЃЦрЃюрЃўрЃЎрЃБрЃарЃљрЃЊ рЃќрЃБрЃАрЃбрЃљрЃЊ, рЃАрЃўрЃдрЃарЃЏрЃўрЃЌ рЃЊрЃљ рЃЏрЃљрЃњрЃљрЃџрЃўрЃЌрЃћрЃЉрЃўрЃЌ.

---

# ­ЪД▒ рЃЋрЃўрЃарЃбрЃБрЃљрЃџрЃўрЃќрЃљрЃфрЃўрЃљ vs рЃЎрЃЮрЃюрЃбрЃћрЃўрЃюрЃћрЃарЃўрЃќрЃљрЃфрЃўрЃљ Рђћ рЃАрЃарЃБрЃџрЃў рЃљрЃ«рЃАрЃюрЃљ рЃЊрЃћрЃЋрЃћрЃџрЃЮрЃърЃћрЃарЃўрЃАрЃЌрЃЋрЃўрЃА

---

## 1№ИЈРЃБ рЃарЃљ рЃљрЃарЃўрЃА **VM (Virtual Machine)**

**VM = рЃАрЃарЃБрЃџрЃў рЃЎрЃЮрЃЏрЃърЃўрЃБрЃбрЃћрЃарЃў рЃАрЃ«рЃЋрЃљ рЃЎрЃЮрЃЏрЃърЃўрЃБрЃбрЃћрЃарЃерЃў.**

­ЪЊд рЃерЃћрЃўрЃфрЃљрЃЋрЃА:

- рЃЏрЃЌрЃћрЃџ **рЃЮрЃърЃћрЃарЃљрЃфрЃўрЃБрЃџ рЃАрЃўрЃАрЃбрЃћрЃЏрЃљрЃА** (рЃЏрЃљрЃњ. Ubuntu, Windows)
    
- рЃљрЃърЃџрЃўрЃЎрЃљрЃфрЃўрЃљрЃА рЃЊрЃљ рЃЏрЃўрЃА рЃДрЃЋрЃћрЃџрЃљ dependency-рЃА
    
- рЃЋрЃўрЃарЃбрЃБрЃљрЃџрЃБрЃа рЃЊрЃўрЃАрЃЎрЃА, RAM-рЃА, CPU-рЃА
    

­ЪДа рЃЏрЃБрЃерЃљрЃЮрЃЉрЃА **рЃ░рЃўрЃърЃћрЃарЃЋрЃўрЃќрЃЮрЃарЃўрЃА** (Hypervisor) рЃќрЃћрЃЏрЃЮрЃЌ Рђћ VMware, Hyper-V, Proxmox, VirtualBoxРђд

```
[ Hardware ]
   РєЊ
[ Hypervisor ]
   РєЊ
[ Guest OS #1 ]  Рєњ Ubuntu Server + .NET API
[ Guest OS #2 ]  Рєњ Windows Server + SQL Server
```

­ЪЉЅ рЃљрЃюрЃБ рЃЌрЃўрЃЌрЃЮ VM-рЃў рЃЌрЃљрЃЋрЃўрЃАрЃў РђюрЃЏрЃўрЃюрЃў-рЃАрЃћрЃарЃЋрЃћрЃарЃўрЃљРђЮ Рђћ  
рЃерЃћрЃю рЃАрЃарЃБрЃџрЃљрЃЊ рЃЏрЃљрЃарЃЌрЃљрЃЋ Kernel-рЃА, User-рЃА, Services-рЃА.

---

## 2№ИЈРЃБ рЃарЃљ рЃљрЃарЃўрЃА **Container**

**Container = рЃўрЃќрЃЮрЃџрЃўрЃарЃћрЃЉрЃБрЃџрЃў рЃърЃарЃЮрЃфрЃћрЃАрЃћрЃЉрЃў рЃўрЃЏрЃљрЃЋрЃћ Kernel-рЃќрЃћ.**

­ЪЊд рЃерЃћрЃўрЃфрЃљрЃЋрЃА:

- рЃљрЃърЃџрЃўрЃЎрЃљрЃфрЃўрЃљрЃА
    
- рЃЏрЃўрЃА dependency-рЃћрЃЉрЃА
    
- рЃЏрЃфрЃўрЃарЃћ runtime рЃњрЃљрЃарЃћрЃЏрЃЮрЃА
    

рЃЏрЃљрЃњрЃарЃљрЃЏ РЮї **рЃљрЃа рЃерЃћрЃўрЃфрЃљрЃЋрЃА рЃАрЃарЃБрЃџ рЃЮрЃърЃћрЃарЃљрЃфрЃўрЃБрЃџ рЃАрЃўрЃАрЃбрЃћрЃЏрЃљрЃА.**

­ЪДа рЃЏрЃБрЃерЃљрЃЮрЃЉрЃА **рЃ░рЃЮрЃАрЃб OS-рЃўрЃА Kernel-рЃќрЃћ**, рЃўрЃќрЃЮрЃџрЃўрЃарЃћрЃЉрЃБрЃџрЃў namespaces-рЃўрЃЌ рЃЊрЃљ cgroup-рЃћрЃЉрЃўрЃЌ.

```
[ Hardware ]
   РєЊ
[ Host OS (Linux) ]
   РєЊ
[ Docker Engine ]
   РєЊ
[ Container #1 ] Рєњ .NET API
[ Container #2 ] Рєњ PostgreSQL
[ Container #3 ] Рєњ Redis
```

­ЪЉЅ рЃДрЃЋрЃћрЃџрЃљ рЃЎрЃЮрЃюрЃбрЃћрЃўрЃюрЃћрЃарЃў рЃўрЃДрЃћрЃюрЃћрЃЉрЃА рЃћрЃарЃЌрЃАрЃљ рЃЊрЃљ рЃўрЃЏрЃљрЃЋрЃћ Kernel-рЃА,  
рЃЏрЃљрЃњрЃарЃљрЃЏ рЃЌрЃўрЃЌрЃЦрЃЮрЃА рЃЊрЃљрЃЏрЃЮрЃБрЃЎрЃўрЃЊрЃћрЃЉрЃћрЃџрЃў рЃърЃарЃЮрЃфрЃћрЃАрЃћрЃЉрЃўрЃљ.

---

## 3№ИЈРЃБ VM vs Container Рђћ рЃФрЃўрЃарЃўрЃЌрЃљрЃЊрЃў рЃњрЃљрЃюрЃАрЃ«рЃЋрЃљрЃЋрЃћрЃЉрЃљ

|рЃЏрЃљрЃ«рЃљрЃАрЃўрЃљрЃЌрЃћрЃЉрЃћрЃџрЃў|VM|Container|
|---|---|---|
|рЃЮрЃърЃћрЃарЃљрЃфрЃўрЃБрЃџрЃў рЃАрЃўрЃАрЃбрЃћрЃЏрЃљ|рЃЌрЃљрЃЋрЃўрЃАрЃў рЃАрЃљрЃЎрЃБрЃЌрЃљрЃарЃў|рЃ░рЃЮрЃАрЃбрЃќрЃћ рЃњрЃљрЃќрЃўрЃљрЃарЃћрЃЉрЃБрЃџрЃў Kernel|
|рЃќрЃЮрЃЏрЃљ|рЃЏрЃФрЃўрЃЏрЃћ (GB-рЃћрЃЉрЃерЃў)|рЃЏрЃАрЃБрЃЉрЃБрЃЦрЃў (MB-рЃћрЃЉрЃерЃў)|
|рЃЕрЃљрЃбрЃЋрЃўрЃарЃЌрЃЋрЃўрЃА рЃЊрЃарЃЮ|рЃгрЃљрЃЏрЃћрЃЉрЃў-рЃгрЃБрЃЌрЃћрЃЉрЃў|рЃЏрЃўрЃџрЃўрЃгрЃљрЃЏрЃћрЃЉрЃў|
|рЃўрЃќрЃЮрЃџрЃљрЃфрЃўрЃљ|рЃАрЃарЃБрЃџрЃў, рЃ░рЃљрЃарЃЊрЃЋрЃћрЃарЃБрЃџрЃў|рЃърЃарЃЮрЃфрЃћрЃАрЃўрЃА рЃЊрЃЮрЃюрЃћрЃќрЃћ|
|рЃБрЃАрЃљрЃцрЃарЃЌрЃ«рЃЮрЃћрЃЉрЃљ|рЃЏрЃљрЃдрЃљрЃџрЃў|рЃерЃћрЃЊрЃљрЃарЃћрЃЉрЃўрЃЌ рЃЊрЃљрЃЉрЃљрЃџрЃў (рЃњрЃљрЃќрЃўрЃљрЃарЃћрЃЉрЃБрЃџрЃў Kernel)|
|рЃърЃЮрЃарЃбрЃљрЃЉрЃћрЃџрЃБрЃарЃЮрЃЉрЃљ|рЃБрЃцрЃарЃЮ рЃарЃЌрЃБрЃџрЃў|рЃФрЃљрЃџрЃўрЃљрЃю рЃЏрЃљрЃарЃбрЃўрЃЋрЃў|
|рЃбрЃўрЃърЃўрЃБрЃарЃў рЃњрЃљрЃЏрЃЮрЃДрЃћрЃюрЃћрЃЉрЃљ|рЃАрЃћрЃарЃЋрЃћрЃарЃћрЃЉрЃўрЃА рЃљрЃю рЃўрЃюрЃцрЃарЃљрЃАрЃбрЃарЃБрЃЦрЃбрЃБрЃарЃўрЃА рЃњрЃљрЃЏрЃўрЃ»рЃЋрЃюрЃљ|рЃЏрЃўрЃЎрЃарЃЮрЃАрЃћрЃарЃЋрЃўрЃАрЃћрЃЉрЃў, DevOps, CI/CD|
|рЃЏрЃљрЃњрЃљрЃџрЃўрЃЌрЃў|VMware, Hyper-V|Docker, Podman, Kubernetes|

---

## 4№ИЈРЃБ рЃарЃљрЃбрЃЮрЃЏ рЃЏрЃўрЃЊрЃўрЃљрЃю рЃЏрЃљрЃўрЃюрЃф VM рЃњрЃќрЃўрЃЌ, рЃарЃЮрЃфрЃљ Docker рЃљрЃарЃАрЃћрЃЉрЃЮрЃЉрЃА?

рЃФрЃљрЃџрЃўрЃљрЃю рЃЏрЃюрЃўрЃерЃЋрЃюрЃћрЃџрЃЮрЃЋрЃљрЃюрЃў рЃЎрЃўрЃЌрЃ«рЃЋрЃљ ­ЪЉЄ

рЃЏрЃўрЃБрЃ«рЃћрЃЊрЃљрЃЋрЃљрЃЊ рЃўрЃЏрЃўрЃАрЃљ, рЃарЃЮрЃЏ Docker рЃЉрЃћрЃЋрЃарЃљрЃЊ рЃАрЃгрЃарЃљрЃцрЃў рЃЊрЃљ рЃЏрЃАрЃБрЃЉрЃБрЃЦрЃўрЃљ,  
**VM рЃ»рЃћрЃа рЃЎрЃўрЃЊрЃћрЃЋ рЃљрЃБрЃфрЃўрЃџрЃћрЃЉрЃћрЃџрЃўрЃљ** рЃњрЃљрЃарЃЎрЃЋрЃћрЃБрЃџ use case-рЃћрЃЉрЃерЃў Рђћ рЃЊрЃљ рЃљрЃў рЃарЃљрЃбрЃЮрЃЏ:

---

### ­ЪДЕ 1. **рЃБрЃАрЃљрЃцрЃарЃЌрЃ«рЃЮрЃћрЃЉрЃљ рЃЊрЃљ рЃАрЃарЃБрЃџрЃў рЃўрЃќрЃЮрЃџрЃљрЃфрЃўрЃљ**

Docker-рЃерЃў рЃДрЃЋрЃћрЃџрЃљ рЃЎрЃЮрЃюрЃбрЃћрЃўрЃюрЃћрЃарЃў рЃћрЃарЃЌ Kernel-рЃА рЃўрЃДрЃћрЃюрЃћрЃЉрЃА.  
рЃЌрЃБ Kernel-рЃќрЃћ рЃўрЃърЃЮрЃЋрЃћрЃА рЃ«рЃљрЃарЃЋрЃћрЃќрЃў Рђћ рЃДрЃЋрЃћрЃџрЃљ рЃЎрЃЮрЃюрЃбрЃћрЃўрЃюрЃћрЃарЃў рЃАрЃљрЃерЃўрЃерЃарЃЮрЃћрЃЉрЃљрЃерЃўрЃљ.

VM-рЃА рЃЌрЃљрЃЋрЃўрЃАрЃў Kernel рЃљрЃЦрЃЋрЃА Рєњ рЃћрЃарЃЌрЃў VM-рЃўрЃА рЃњрЃљрЃбрЃћрЃ«рЃЋрЃљ рЃАрЃ«рЃЋрЃљрЃќрЃћ рЃњрЃљрЃЋрЃџрЃћрЃюрЃљрЃА рЃљрЃа рЃљрЃ«рЃЊрЃћрЃюрЃА.

> ­ЪЊї рЃЏрЃљрЃњрЃљрЃџрЃўрЃЌрЃљрЃЊ: рЃЉрЃљрЃюрЃЎрЃћрЃЉрЃў, рЃАрЃљрЃЏрЃЌрЃљрЃЋрЃарЃЮрЃЉрЃЮ рЃАрЃћрЃарЃЋрЃћрЃарЃћрЃЉрЃў, SCADA рЃАрЃўрЃАрЃбрЃћрЃЏрЃћрЃЉрЃў Рђћ VM-рЃћрЃЉрЃќрЃћ рЃљрЃЏрЃБрЃерЃљрЃЋрЃћрЃЉрЃћрЃю.

---

### ­ЪДЕ 2. **рЃАрЃарЃБрЃџрЃў рЃЮрЃърЃћрЃарЃљрЃфрЃўрЃБрЃџрЃў рЃАрЃўрЃАрЃбрЃћрЃЏрЃўрЃА рЃАрЃљрЃГрЃўрЃарЃЮрЃћрЃЉрЃљ**

рЃЌрЃБ рЃљрЃърЃџрЃўрЃЎрЃљрЃфрЃўрЃљ рЃЏрЃЮрЃўрЃЌрЃ«рЃЮрЃЋрЃА Windows-рЃА рЃљрЃю рЃАрЃърЃћрЃфрЃўрЃљрЃџрЃБрЃа Linux Kernel рЃЏрЃЮрЃЊрЃБрЃџрЃА  
(рЃЏрЃљрЃњ. рЃЊрЃарЃљрЃўрЃЋрЃћрЃарЃћрЃЉрЃў, device access, firewall modules) Рђћ  
рЃћрЃА Docker-рЃерЃў рЃерЃћрЃБрЃФрЃџрЃћрЃЉрЃћрЃџрЃўрЃљ.

> ­ЪЊї рЃЏрЃљрЃњрЃљрЃџрЃўрЃЌрЃљрЃЊ: VPN gateways, Security appliances, Network firewalls Рђћ рЃЏрЃБрЃерЃљрЃЮрЃЉрЃћрЃю VM-рЃћрЃЉрЃљрЃЊ.

---

### ­ЪДЕ 3. **рЃўрЃюрЃцрЃарЃљрЃАрЃбрЃарЃБрЃЦрЃбрЃБрЃарЃўрЃА рЃўрЃќрЃЮрЃџрЃљрЃфрЃўрЃљ**

VM рЃњрЃљрЃЏрЃЮрЃўрЃДрЃћрЃюрЃћрЃЉрЃљ рЃарЃЮрЃфрЃљ рЃБрЃюрЃЊрЃљ РђърЃцрЃўрЃќрЃўрЃЎрЃБрЃарЃў рЃАрЃћрЃарЃЋрЃћрЃарЃўрЃА рЃЏрЃАрЃњрЃљрЃЋрЃАрЃљрЃЊРђю рЃњрЃљрЃљрЃюрЃљрЃгрЃўрЃџрЃЮ рЃњрЃљрЃарЃћрЃЏрЃЮрЃћрЃЉрЃў:

- Dev / Test / Staging / Prod
    
- рЃАрЃ«рЃЋрЃљрЃЊрЃљрЃАрЃ«рЃЋрЃљ рЃњрЃБрЃюрЃЊрЃћрЃЉрЃўрЃАрЃЌрЃЋрЃўрЃА рЃњрЃљрЃЏрЃЮрЃДрЃЮрЃцрЃўрЃџрЃў рЃњрЃљрЃарЃћрЃЏрЃЮрЃћрЃЉрЃў
    

> рЃЌрЃўрЃЌрЃЮ рЃњрЃБрЃюрЃЊрЃА рЃЌрЃљрЃЋрЃўрЃАрЃў VM рЃљрЃЦрЃЋрЃА Рєњ рЃАрЃљрЃЎрЃБрЃЌрЃљрЃарЃў root access, рЃАрЃљрЃЎрЃБрЃЌрЃљрЃарЃў network, рЃЊрЃљ рЃАрЃ«рЃЋрЃљрЃќрЃћ рЃњрЃљрЃЋрЃџрЃћрЃюрЃљ рЃљрЃа рЃљрЃЦрЃЋрЃА.

---

### ­ЪДЕ 4. **рЃЏрЃарЃљрЃЋрЃљрЃџрЃЮрЃърЃћрЃарЃљрЃфрЃўрЃБрЃџрЃў рЃњрЃљрЃарЃћрЃЏрЃЮ**

Docker рЃЏрЃ«рЃЮрЃџрЃЮрЃЊ рЃўрЃЏрЃљрЃЋрЃћ Kernel-рЃќрЃћ рЃЏрЃБрЃерЃљрЃЮрЃЉрЃА:

- рЃЋрЃћрЃа рЃњрЃљрЃБрЃерЃЋрЃћрЃЉ Windows container-рЃА Linux-рЃќрЃћ (рЃЊрЃљ рЃърЃўрЃарЃўрЃЦрЃўрЃЌ)
    

VM-рЃўрЃЌ рЃерЃћрЃњрЃўрЃФрЃџрЃўрЃљ:

- Windows-рЃќрЃћ рЃњрЃљрЃБрЃерЃЋрЃљ Ubuntu VM
    
- Linux-рЃќрЃћ рЃњрЃљрЃБрЃерЃЋрЃљ Windows Server VM
    

---

### ­ЪДЕ 5. **рЃўрЃюрЃцрЃарЃљрЃАрЃбрЃарЃБрЃЦрЃбрЃБрЃарЃўрЃА рЃ░рЃЮрЃАрЃбрЃўрЃюрЃњрЃў рЃЊрЃљ Cloud Providers**

Cloud-рЃАрЃћрЃарЃЋрЃўрЃАрЃћрЃЉрЃў (AWS, Azure, GCP) VM-рЃћрЃЉрЃќрЃћрЃљ рЃљрЃерЃћрЃюрЃћрЃЉрЃБрЃџрЃў.  
рЃЎрЃџрЃўрЃћрЃюрЃбрЃА VM рЃљрЃФрЃџрЃћрЃЋрЃћрЃю (EC2, Azure VM),  
рЃЊрЃљ рЃўрЃА рЃерЃўрЃњрЃюрЃўрЃЌ рЃБрЃЎрЃЋрЃћ рЃЊрЃћрЃЉрЃА Docker-рЃћрЃЉрЃА.

> ­ЪЊї рЃљрЃюрЃБ: **рЃЎрЃЮрЃюрЃбрЃћрЃўрЃюрЃћрЃарЃћрЃЉрЃў рЃ«рЃерЃўрЃарЃљрЃЊ VM-рЃерЃў рЃЏрЃБрЃерЃљрЃЮрЃЉрЃћрЃю.**

---

## 5№ИЈРЃБ рЃарЃћрЃљрЃџрЃБрЃарЃў рЃърЃарЃљрЃЦрЃбрЃўрЃЎрЃљ Рђћ VM + Container рЃћрЃарЃЌрЃљрЃЊ

рЃЌрЃљрЃюрЃљрЃЏрЃћрЃЊрЃарЃЮрЃЋрЃћ рЃљрЃарЃЦрЃўрЃбрЃћрЃЦрЃбрЃБрЃарЃљрЃерЃў рЃЕрЃЋрЃћрЃБрЃџрЃћрЃЉрЃарЃўрЃЋрЃўрЃљ рЃљрЃАрЃћрЃЌрЃў рЃЏрЃЮрЃЊрЃћрЃџрЃў ­ЪЉЄ

```
[ рЃцрЃўрЃќрЃўрЃЎрЃБрЃарЃў рЃАрЃћрЃарЃЋрЃћрЃарЃў ]
   РєЊ
[ Hypervisor ]
   РєЊ
[ VM #1 - Kubernetes Node ]
   РєЊ
[ Docker Containers (.NET API, Redis, PostgreSQL...) ]
```

­ЪДа рЃерЃћрЃЊрЃћрЃњрЃў:

- VM рЃљрЃФрЃџрЃћрЃЋрЃА рЃБрЃАрЃљрЃцрЃарЃЌрЃ«рЃЮрЃћрЃЉрЃљрЃА рЃЊрЃљ рЃўрЃюрЃцрЃарЃљрЃАрЃбрЃарЃБрЃЦрЃбрЃБрЃарЃБрЃџ рЃўрЃќрЃЮрЃџрЃљрЃфрЃўрЃљрЃА
    
- Docker рЃљрЃФрЃџрЃћрЃЋрЃА рЃАрЃўрЃАрЃгрЃарЃљрЃцрЃћрЃА рЃЊрЃљ рЃЏрЃЮрЃЦрЃюрЃўрЃџрЃЮрЃЉрЃљрЃА
    

> рЃљрЃюрЃБ VM + Docker = рЃўрЃЊрЃћрЃљрЃџрЃБрЃарЃў рЃЎрЃЮрЃЏрЃЉрЃЮ ­ЪДЕ

## ­Ъћџ рЃерЃћрЃ»рЃљрЃЏрЃћрЃЉрЃљ

> **VM** = рЃАрЃарЃБрЃџрЃў рЃЎрЃЮрЃЏрЃърЃўрЃБрЃбрЃћрЃарЃў  
> **Container** = рЃўрЃќрЃЮрЃџрЃўрЃарЃћрЃЉрЃБрЃџрЃў рЃърЃарЃЮрЃфрЃћрЃАрЃўрЃА рЃњрЃљрЃарЃћрЃЏрЃЮ

­ЪДа **VM-рЃўрЃА рЃБрЃърЃўрЃарЃљрЃбрЃћрЃАрЃЮрЃЉрЃљ:** рЃАрЃарЃБрЃџрЃў рЃўрЃќрЃЮрЃџрЃљрЃфрЃўрЃљ, рЃБрЃАрЃљрЃцрЃарЃЌрЃ«рЃЮрЃћрЃЉрЃљ, рЃЏрЃарЃљрЃЋрЃљрЃџрЃЮрЃърЃћрЃарЃљрЃфрЃўрЃБрЃџрЃў рЃЏрЃ«рЃљрЃарЃЊрЃљрЃГрЃћрЃарЃљ.  
РџА **Container-рЃўрЃА рЃБрЃърЃўрЃарЃљрЃбрЃћрЃАрЃЮрЃЉрЃљ:** рЃАрЃўрЃАрЃгрЃарЃљрЃцрЃћ, рЃЏрЃЮрЃЦрЃюрЃўрЃџрЃЮрЃЉрЃљ, рЃЏрЃАрЃБрЃЉрЃБрЃЦрЃЮрЃЉрЃљ, CI/CD рЃЊрЃљ scale.

рЃЊрЃљ рЃЊрЃдрЃћрЃА рЃЌрЃўрЃЌрЃЦрЃЏрЃўрЃА рЃДрЃЋрЃћрЃџрЃљ рЃАрЃћрЃарЃўрЃЮрЃќрЃБрЃџрЃў рЃАрЃўрЃАрЃбрЃћрЃЏрЃљ рЃљрЃћрЃарЃЌрЃўрЃљрЃюрЃћрЃЉрЃА рЃЮрЃарЃА:

> РђъVM-рЃќрЃћ рЃњрЃљрЃерЃЋрЃћрЃЉрЃБрЃџрЃў Kubernetes node-рЃћрЃЉрЃў, рЃарЃЮрЃЏрЃџрЃћрЃЉрЃўрЃф рЃЏрЃљрЃарЃЌрЃљрЃЋрЃћрЃю Docker рЃЎрЃЮрЃюрЃбрЃћрЃўрЃюрЃћрЃарЃћрЃЉрЃА.Рђю
