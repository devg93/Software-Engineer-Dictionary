Fault Tolerance-рЃўрЃА (рЃерЃћрЃфрЃЊрЃЮрЃЏрЃљрЃњрЃљрЃЏрЃФрЃџрЃћрЃЮрЃЉрЃўрЃА) **рЃАрЃбрЃарЃљрЃбрЃћрЃњрЃўрЃћрЃЉрЃў** Рђћ рЃћрЃА рЃљрЃарЃўрЃА рЃЏрЃћрЃЌрЃЮрЃЊрЃћрЃЉрЃў, рЃарЃЮрЃЏрЃџрЃўрЃЌрЃљрЃф рЃАрЃўрЃАрЃбрЃћрЃЏрЃљ _рЃљрЃњрЃарЃФрЃћрЃџрЃћрЃЉрЃА рЃЏрЃБрЃерЃљрЃЮрЃЉрЃљрЃА_ рЃЏрЃљрЃарЃфрЃ«рЃўрЃАрЃљрЃА.  
рЃўрЃАрЃўрЃюрЃў рЃўрЃДрЃЮрЃцрЃљ рЃърЃарЃЮрЃњрЃарЃљрЃЏрЃБрЃџ, рЃўрЃюрЃцрЃарЃљрЃАрЃбрЃарЃБрЃЦрЃбрЃБрЃарЃБрЃџ рЃЊрЃљ рЃЏрЃЮрЃюрЃљрЃфрЃћрЃЏрЃБрЃа рЃЊрЃЮрЃюрЃћрЃћрЃЉрЃљрЃЊ.

---

## РџЎ№ИЈ рЃФрЃўрЃарЃўрЃЌрЃљрЃЊрЃў Fault Tolerance рЃАрЃбрЃарЃљрЃбрЃћрЃњрЃўрЃћрЃЉрЃў

### 1. **Redundancy (рЃАрЃљрЃарЃћрЃќрЃћрЃарЃЋрЃЮ рЃарЃћрЃАрЃБрЃарЃАрЃћрЃЉрЃў)**

- рЃДрЃЋрЃћрЃџрЃљрЃќрЃћ рЃцрЃБрЃюрЃЊрЃљрЃЏрЃћрЃюрЃбрЃБрЃарЃў рЃърЃарЃўрЃюрЃфрЃўрЃърЃў: рЃерЃћрЃЦрЃЏрЃћрЃюрЃў рЃљрЃАрЃџрЃћрЃЉрЃў.
    
- рЃерЃћрЃўрЃФрЃџрЃћрЃЉрЃљ рЃўрЃДрЃЮрЃА:
    
    - **Hardware redundancy** РђЊ рЃЮрЃарЃў рЃАрЃћрЃарЃЋрЃћрЃарЃў рЃљрЃю node.
        
    - **Service redundancy** РђЊ рЃЏрЃарЃљрЃЋрЃљрЃџрЃў API instance (Kubernetes replicas).
        
    - **Network redundancy** РђЊ backup connection рЃљрЃю DNS failover.
        
- ­ЪЉЅ рЃЏрЃўрЃќрЃљрЃюрЃў: рЃћрЃарЃЌрЃўрЃА рЃЕрЃљрЃЋрЃљрЃарЃЊрЃюрЃўрЃАрЃљрЃА рЃАрЃ«рЃЋрЃљрЃЏ рЃўрЃЏрЃБрЃерЃљрЃЮрЃА рЃЊрЃљрЃБрЃДрЃЮрЃЋрЃюрЃћрЃЉрЃџрЃўрЃЋ.
    

---

### 2. **Replication**

- рЃЏрЃЮрЃюрЃљрЃфрЃћрЃЏрЃћрЃЉрЃўрЃА рЃљрЃАрЃџрЃћрЃЉрЃўрЃА рЃерЃћрЃюрЃљрЃ«рЃЋрЃљ рЃАрЃ«рЃЋрЃљрЃЊрЃљрЃАрЃ«рЃЋрЃљ рЃџрЃЮрЃЎрЃљрЃфрЃўрЃљрЃерЃў.
    
- рЃбрЃўрЃърЃћрЃЉрЃў:
    
    - **Database replication** (SQL AlwaysOn, PostgreSQL streaming replication).
        
    - **Message replication** (Kafka replicas, RabbitMQ HA queues).
        
    - **Cache replication** (Redis cluster).
        
- ­ЪЉЅ рЃБрЃќрЃарЃБрЃюрЃЋрЃћрЃџрЃДрЃЮрЃцрЃА рЃЏрЃЮрЃюрЃљрЃфрЃћрЃЏрЃЌрЃљ рЃ«рЃћрЃџрЃЏрЃўрЃАрЃљрЃгрЃЋрЃЊрЃЮрЃЏрЃЮрЃЉрЃљрЃА node-рЃўрЃА рЃЏрЃљрЃарЃфрЃ«рЃўрЃАрЃљрЃА.
    

---

### 3. **Failover Strategy**

- рЃљрЃЋрЃбрЃЮрЃЏрЃљрЃбрЃБрЃарЃў рЃњрЃљрЃЊрЃљрЃарЃЌрЃЋрЃљ рЃФрЃўрЃарЃўрЃЌрЃљрЃЊрЃў рЃарЃћрЃАрЃБрЃарЃАрЃўрЃЊрЃљрЃю рЃАрЃљрЃарЃћрЃќрЃћрЃарЃЋрЃЮрЃќрЃћ.
    
- рЃбрЃўрЃърЃћрЃЉрЃў:
    
    - **ActiveРђЊPassive** Рєњ рЃћрЃарЃЌрЃў рЃЏрЃБрЃерЃљрЃЮрЃЉрЃА, рЃЏрЃћрЃЮрЃарЃћ standby.
        
    - **ActiveРђЊActive** Рєњ рЃЮрЃарЃўрЃЋрЃћ рЃЏрЃБрЃерЃљрЃЮрЃЉрЃА рЃЊрЃљ Load Balancer рЃљрЃюрЃљрЃгрЃўрЃџрЃћрЃЉрЃА рЃбрЃарЃљрЃцрЃўрЃЎрЃА.
        
- ­ЪЉЅ downtime рЃърЃарЃљрЃЦрЃбрЃўрЃЎрЃБрЃџрЃљрЃЊ рЃюрЃБрЃџрЃЮрЃЋрЃљрЃюрЃўрЃљ.
    

---

### 4. **Graceful Degradation**

- рЃАрЃўрЃАрЃбрЃћрЃЏрЃљ рЃљрЃњрЃарЃФрЃћрЃџрЃћрЃЉрЃА рЃюрЃљрЃгрЃўрЃџрЃЮрЃЉрЃарЃўрЃЋ рЃцрЃБрЃюрЃЦрЃфрЃўрЃЮрЃюрЃўрЃарЃћрЃЉрЃљрЃА рЃърЃарЃЮрЃЉрЃџрЃћрЃЏрЃўрЃА рЃЊрЃарЃЮрЃА.  
    рЃЏрЃљрЃњ.: рЃњрЃљрЃЊрЃљрЃ«рЃЊрЃўрЃА рЃАрЃћрЃарЃЋрЃўрЃАрЃў рЃњрЃљрЃЕрЃћрЃарЃЊрЃљ Рђћ рЃљрЃърЃџрЃўрЃЎрЃљрЃфрЃўрЃљ рЃЏрЃљрЃўрЃюрЃф рЃљрЃЕрЃЋрЃћрЃюрЃћрЃЉрЃА рЃърЃарЃЮрЃЊрЃБрЃЦрЃбрЃўрЃА рЃЎрЃљрЃбрЃљрЃџрЃЮрЃњрЃА.
    
- ­ЪЉЅ рЃЏрЃЮрЃЏрЃ«рЃЏрЃљрЃарЃћрЃЉрЃћрЃџрЃў рЃљрЃа рЃЎрЃљрЃарЃњрЃљрЃЋрЃА рЃЏрЃЌрЃџрЃўрЃљрЃюрЃљрЃЊ рЃгрЃЋрЃЊрЃЮрЃЏрЃљрЃА.
    

---

### 5. **Retry / Backoff Strategy**

- рЃерЃћрЃфрЃЊрЃЮрЃЏрЃўрЃАрЃљрЃА рЃЏрЃЮрЃЌрЃ«рЃЮрЃЋрЃюрЃљ рЃЏрЃћрЃЮрЃарЃЊрЃћрЃЉрЃљ рЃњрЃљрЃарЃЎрЃЋрЃћрЃБрЃџрЃў рЃгрЃћрЃАрЃўрЃЌ:
    
    - **Exponential backoff** РђЊ рЃДрЃЮрЃЋрЃћрЃџ рЃ»рЃћрЃарЃќрЃћ рЃўрЃќрЃарЃЊрЃћрЃЉрЃљ Retry рЃўрЃюрЃбрЃћрЃарЃЋрЃљрЃџрЃў.
        
    - **Jitter** РђЊ рЃЊрЃљрЃЏрЃљрЃбрЃћрЃЉрЃўрЃЌрЃў рЃерЃћрЃЏрЃЌрЃ«рЃЋрЃћрЃЋрЃўрЃЌрЃў рЃЊрЃарЃЮ РђюрЃЊрЃљрЃФрЃљрЃЉрЃБрЃџрЃЮрЃЉрЃўрЃАРђЮ рЃЌрЃљрЃЋрЃўрЃЊрЃљрЃю рЃљрЃАрЃљрЃфрЃўрЃџрЃћрЃЉрЃџрЃљрЃЊ.
        
- ­ЪЉЅ Polly-рЃерЃў рЃљрЃю Resilience pipeline-рЃерЃў рЃ«рЃерЃўрЃарЃљрЃЊ рЃњрЃљрЃЏрЃЮрЃўрЃДрЃћрЃюрЃћрЃЉрЃљ.
    

```csharp
var policy = Policy
    .Handle<Exception>()
    .WaitAndRetryAsync(3, retry => TimeSpan.FromSeconds(Math.Pow(2, retry)));
```

---

### 6. **Circuit Breaker Pattern**

- рЃарЃЮрЃЊрЃћрЃАрЃљрЃф рЃАрЃћрЃарЃЋрЃўрЃАрЃў рЃ«рЃерЃўрЃарЃљрЃЊ рЃљрЃњрЃЊрЃћрЃЉрЃА рЃерЃћрЃфрЃЊрЃЮрЃЏрЃљрЃА Рђћ рЃЏрЃЮрЃЌрЃ«рЃЮрЃЋрЃюрЃћрЃЉрЃў рЃЊрЃарЃЮрЃћрЃЉрЃўрЃЌ рЃўрЃЉрЃџрЃЮрЃЎрЃћрЃЉрЃљ.
    
- рЃЌрЃљрЃЋрЃўрЃЊрЃљрЃю рЃўрЃфрЃўрЃџрЃћрЃЉрЃА cascading failure-рЃА рЃЏрЃЌрЃћрЃџ рЃАрЃўрЃАрЃбрЃћрЃЏрЃљрЃерЃў.
    
- ­ЪЉЅ Polly рЃљрЃю Dapr Resiliency policies-рЃерЃў рЃДрЃЋрЃћрЃџрЃљрЃќрЃћ рЃњрЃљрЃЋрЃарЃфрЃћрЃџрЃћрЃЉрЃБрЃџрЃўрЃљ.
    

---

### 7. **Bulkhead Isolation**

- рЃАрЃўрЃАрЃбрЃћрЃЏрЃўрЃА рЃюрЃљрЃгрЃўрЃџрЃћрЃЉрЃўрЃА РђюрЃњрЃљрЃДрЃЮрЃцрЃљРђЮ рЃўрЃАрЃћ, рЃарЃЮрЃЏ рЃћрЃарЃЌрЃў рЃЏрЃљрЃарЃфрЃ«рЃў рЃљрЃа рЃЊрЃљрЃљрЃюрЃњрЃарЃўрЃЮрЃА рЃАрЃ«рЃЋрЃљ.  
    рЃЏрЃљрЃњ.: рЃЌрЃўрЃЌрЃЮрЃћрЃБрЃџрЃў module (Payments, Notifications, Users) рЃЏрЃБрЃерЃљрЃЮрЃЉрЃА рЃЊрЃљрЃЏрЃЮрЃБрЃЎрЃўрЃЊрЃћрЃЉрЃџрЃљрЃЊ thread pool-рЃќрЃћ.
    
- ­ЪЉЅ рЃЌрЃљрЃЋрЃўрЃЊрЃљрЃю рЃљрЃфрЃўрЃџрЃћрЃЉрЃА resource exhaustion-рЃА.
    

---

### 8. **Load Balancing**

- рЃбрЃарЃљрЃцрЃўрЃЎрЃўрЃА рЃЌрЃљрЃюрЃљрЃЉрЃарЃљрЃЊ рЃњрЃљрЃЊрЃљрЃюрЃљрЃгрЃўрЃџрЃћрЃЉрЃљ рЃЏрЃарЃљрЃЋрЃљрЃџрЃў instance-рЃќрЃћ.
    
- рЃерЃћрЃўрЃФрЃџрЃћрЃЉрЃљ рЃўрЃДрЃЮрЃА L4/L7 рЃЊрЃЮрЃюрЃћрЃќрЃћ (HAProxy, Nginx, Azure Load Balancer).
    
- ­ЪЉЅ рЃћрЃарЃЌ node-рЃќрЃћ рЃњрЃљрЃЊрЃљрЃбрЃЋрЃўрЃарЃЌрЃЋрЃљ рЃљрЃа рЃЕрЃљрЃљрЃњрЃЊрЃћрЃЉрЃА рЃЏрЃЌрЃћрЃџ рЃАрЃўрЃАрЃбрЃћрЃЏрЃљрЃА.
    

---

### 9. **Data Checkpointing & Recovery**

- рЃАрЃўрЃАрЃбрЃћрЃЏрЃўрЃА рЃЏрЃЊрЃњрЃЮрЃЏрЃљрЃарЃћрЃЮрЃЉрЃўрЃА рЃерЃћрЃюрЃљрЃ«рЃЋрЃљ (snapshot).
    
- рЃЕрЃљрЃЋрЃљрЃарЃЊрЃюрЃўрЃАрЃљрЃА рЃљрЃдрЃЊрЃњрЃћрЃЉрЃљ рЃЉрЃЮрЃџрЃЮ рЃЋрЃљрЃџрЃўрЃЊрЃБрЃарЃў рЃЏрЃЊрЃњрЃЮрЃЏрЃљрЃарЃћрЃЮрЃЉрЃўрЃЊрЃљрЃю.
    
- ­ЪЉЅ рЃњрЃљрЃЏрЃЮрЃўрЃДрЃћрЃюрЃћрЃЉрЃљ stream-processing рЃАрЃўрЃАрЃбрЃћрЃЏрЃћрЃЉрЃерЃў (Kafka, Flink, Redis Streams).
    

---

### 10. **Monitoring + Self-Healing**

- Health-check endpoints + Prometheus alerts + Kubernetes auto-restart.
    
- ­ЪЉЅ рЃАрЃўрЃАрЃбрЃћрЃЏрЃўрЃА рЃљрЃЋрЃбрЃЮрЃЏрЃљрЃбрЃБрЃарЃў рЃљрЃдрЃЊрЃњрЃћрЃюрЃљ рЃЊрЃљ рЃЮрЃърЃћрЃарЃљрЃбрЃЮрЃарЃўрЃА рЃерЃћрЃбрЃДрЃЮрЃЉрЃўрЃюрЃћрЃЉрЃљ.

## РюЁ рЃЏрЃЮрЃЎрЃџрЃћ рЃерЃћрЃЏрЃљрЃ»рЃљрЃЏрЃћрЃЉрЃћрЃџрЃў рЃфрЃ«рЃарЃўрЃџрЃў

|рЃАрЃбрЃарЃљрЃбрЃћрЃњрЃўрЃљ|рЃЏрЃўрЃќрЃљрЃюрЃў|рЃЏрЃљрЃњрЃљрЃџрЃўрЃЌрЃў .NET-рЃерЃў|
|---|---|---|
|Redundancy|Backup instance|Multiple Kestrel pods|
